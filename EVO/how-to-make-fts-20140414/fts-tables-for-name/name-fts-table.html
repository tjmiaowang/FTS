nameFtsTable¶

This table contains FTS content for Address.

Member detailes¶

docId:
You do not need to store docId by yourself. It is assigned automatically by SQLite when you insert a record. docId is an alias of SQLite’s rowid.
languageCode:
Language code for this entry. See languageCode for FTS
namedObjectId:

The named object Id of administrative area or road name stored in criterion A to J columns. Set the lowest address level criterion’s named object Id. For example, if criterionA, B, C and D correspond to Province, City, City district and Street, namedObjectId should be stored like the following example.

Example of how to set namedObjectId
namedObjectId criterionA criterionB criterionC criterionD description
10000 广东省       Set criterionA’s named object Id to namedObjectId column
10010 广东省 湛江市     Set criterionB’s named object Id to namedObjectId column
10100 广东省 湛江市 徐闻县   Set criterionC’s named object Id to namedObjectId column
11000 广东省 湛江市 徐闻县 民主路 Set criterionD’s named object Id to namedObjectId column
11001 广东省 湛江市 徐闻县 华建路 Set criterionD’s named object Id to namedObjectId column
mortonCode:
Morton code of the Named Object.
criterionA - criterionJ:
Address names, like ‘Beijing’, ‘Shunyi District’, of this record. See criterion A to J columns to know which address component should be set for each criterion columns
searchTags:

Space separated list of the Named Object’s synonyms. Set the same synonyms with which NVC or SLI has.

In China, if a named object has both simplified and traditional Chinese name, set official one to a criterion column, others to searchTags.

exceptionTags:
Currently there is no requirement for this column.
priorityIndex:
Currently there is no requirement for this column.

Are duplicate namedObjectId allowd?¶

Each FTS record must be unique by languageCode and namedObjectId. Thus, below is valid.

Unique by languageCode and namedObjectId
languageCode namedObjectId criterionX
0 (UNDEFINED_LANGUAGE_CODE) 100 澳门
182 (English) 100 Macau

But the following is not allowed.

Duplicate namedObjectId in a languageCode
languageCode namedObjectId criterionX
0 (UNDEFINED_LANGUAGE_CODE) 100 澳门
0 (UNDEFINED_LANGUAGE_CODE) 100 澳門
182 (English) 100 Macau

The reason of this rule is because if we allow duplicate namedObjectId in a languageCode situation, an application have to scan a result set and remove duplicate one. Apparently, this behavior affects performance, thus should be avoided.
